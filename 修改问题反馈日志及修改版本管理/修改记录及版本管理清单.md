# 修改记录及版本管理清单

> **用途**：统一管理产品迭代过程中的版本日志与修改记录。
>
> **工作流程**：
> 1. 日常修改 → 记录到 [修改问题记录.md](修改问题记录.md)
> 2. 定版时 → 汇总本次版本内容到 [产品版本日志.md](产品版本日志.md)（新增版本条目），同步更新 [修改问题记录.md](修改问题记录.md) 中的版本号
> 3. 用户反馈 TXT 存入本目录，格式：`YYYYMMDD产品设计方案问题反馈.txt`

---

## 文件结构

```
修改问题反馈日志及修改版本管理/
├── 修改记录及版本管理清单.md    ← 本文件（索引导航）
├── 产品版本日志.md              ← 文件1：按版本汇总的修改概述
├── 修改问题记录.md              ← 文件2：按时间/模块的详细修改记录
├── 20260714产品设计方案问题反馈.txt  ← 用户反馈原稿
└── YYYYMMDD产品设计方案问题反馈.txt  ← 后续反馈（按日期命名）
```

## 文件说明

### 文件1：[产品版本日志.md](产品版本日志.md)

| 字段 | 说明 |
|------|------|
| 版本号 | V{模块}.{页面}.{功能点}，如 V1.2.3 |
| 版本修改概述 | 该版本包含的所有修改的高级描述，按模块/页面/功能点三级分类 |

### 文件2：[修改问题记录.md](修改问题记录.md)

| 字段 | 说明 |
|------|------|
| 模块 | 一级菜单所属模块 |
| 功能页面 | 具体页面名称 |
| 修改点描述 | 修改内容概述 |
| 产品变更版本号 | 对应的产品版本号 |

## 版本号规则

```
V 1  .  2  .  3
│ │  │  │  │  └── 功能点调整（字段增删/样式修复/文案/Bug修复）
│ │  │  │  └───── 功能页面调整（新增页面/弹窗/页面布局重大调整）
│ │  └──┴──────── 模块增加/重构（新增或重组一级菜单模块）
└─ 固定前缀
```

| 层级 | 触发条件 | 示例 |
|------|----------|------|
| V**X**.0.0 | 新增或重组一级菜单模块 | 菜单拆分、模块合并 |
| V0.**Y**.0 | 新增页面、新增弹窗、页面重大布局调整 | 新增详情页、新增业务弹窗 |
| V0.0.**Z** | 字段增删、样式修复、文案调整、Bug修复 | 移除字段、按钮样式对齐 |

## 定版操作指引

当告知「修改定版」时，执行以下操作：

1. **读取** [修改问题记录.md](修改问题记录.md) 中本次待定版的修改内容
2. **新增** [产品版本日志.md](产品版本日志.md) 顶部版本条目（版本号 + 概述）
3. **回填** [修改问题记录.md](修改问题记录.md) 中各项的「产品变更版本号」列
4. **清空** 待定版区域，已定版条目保留在已定版表格中
