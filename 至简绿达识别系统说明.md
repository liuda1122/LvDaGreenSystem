# 金融机构"至简绿达"识别系统 · 系统说明

## 系统概述

"至简绿达"是一套面向商业银行的绿色金融智能识别与业务管理系统，覆盖绿色产业洞察、客户营销、项目对接、绿色认定、反洗绿复核等全链路业务场景。系统采用**行内部署**架构，全部数据存储与模型推理均在行内服务器完成，**数据不出行**，满足金融机构数据安全与合规要求。

---

## 政策背景

在中国"双碳"目标（2030 年前碳达峰、2060 年前碳中和）的宏观框架下，绿色金融已成为推动经济社会低碳转型的核心引擎。人民银行、国家金融监督管理总局等主管部门陆续出台了一系列政策，明确要求金融机构建立绿色信贷识别、统计、报送和披露体系，并将绿色金融纳入银行考核评价。

### 核心政策框架

**绿色金融顶层设计**

- **《银行业金融机构绿色金融评价方案》**（人民银行，2021）— 将绿色信贷、绿色债券纳入银行考核评价体系，按季度开展绿色金融评价
- **《关于进一步强化金融支持绿色低碳发展的指导意见》**（人民银行等七部委，2024）— 提出强化绿色金融标准体系建设、推动绿色信贷规模持续增长

**绿色分类与认定标准**

- **《绿色金融支持项目目录》（2025 版）** — 界定绿色产业范围与分类标准，是金融机构开展绿色信贷认定的核心依据，涵盖节能环保、清洁生产、清洁能源、生态环境、基础设施绿色升级、绿色服务六大领域

**转型金融框架**

- **《转型金融支持目录》** — 聚焦火电、钢铁、有色、石化、化工、建材、造纸、航空八大高碳行业的低碳转型，明确转型金融支持范围与标准，为金融机构服务高碳行业转型提供分类依据

**碳排放双控与园区政策**

- **《2030 年前碳达峰行动方案》**（国务院）— 提出"碳排放总量和强度双控"制度，选择 100 个具有典型代表性的城市和园区开展碳达峰试点
- **零碳/低碳园区试点政策** — 鼓励园区分阶段推进低碳园区、近零碳园区、零碳园区建设，覆盖用能结构转型、节能降碳、产业结构优化、资源节约集约、基础设施升级、先进技术应用、能碳管理、改革创新等八大场景
- **"五碳"工程** — 源头减碳、过程降碳、末端固碳、碳资产开发、碳金融服务，构建全链条碳管理体系

**循环经济**

- **《"十四五"循环经济发展规划》** — 推动工业固废综合利用、再生资源回收利用、废旧动力电池回收、再制造、园区循环化改造等重点领域发展

**监管报送与风险防控**

- **绿色贷款专项统计制度（A3327）** — 要求金融机构按季度报送绿色贷款统计数据，涵盖贷款规模、行业分布、环境效益等维度
- **"反洗绿"监管要求** — 防范企业将非绿色项目包装为绿色项目套取绿色信贷优惠，要求金融机构建立绿色资产质量复核验证机制

### 系统与政策对标

| 政策体系 | 系统支撑模块 |
|----------|-------------|
| 双碳目标与绿色金融评价 | 全系统 — 绿色认定 + 效益测算 + 报表报送 |
| 《绿色金融支持项目目录》（2025 版） | 合同绿色认定、贷款预识别 — 智能识别模型核心依据 |
| 转型金融框架 | 转型客户营销、转型产业地图筛选 |
| 碳排放双控 | "碳排放双控"专题项目清单 |
| 零碳/低碳园区试点 | 零碳/低碳园区项目清单（3 阶段 × 8 场景） |
| 循环经济规划 | 循环经济项目清单（6 大领域） |
| A3327 绿色贷款统计 | 报表管理模块 |
| 反洗绿监管 | "反洗绿"模型智能识别 |

## 一、绿色与转型产业地图

以 GIS 地图为核心载体，直观呈现区域绿色产业与转型产业的分布热力，为业务人员提供"一张图"式的产业洞察入口。

### 1.1 绿色产业地图总览

以 GIS 地图展示区域绿色产业分布，按绿色深度分为三个层级：

| 层级 | 定义 | 地图标识 |
|------|------|----------|
| 深绿产业 | 完全符合《绿色金融支持项目目录》的项目/企业 | 深绿色标记 |
| 中绿产业 | 部分符合、具备绿色潜力的项目/企业 | 中绿色标记 |
| 浅绿产业 | 有转型意愿但尚未达标的项目/企业 | 浅绿色标记 |

同时展示**转型产业分布**，标注区域内八大高碳行业中具有低碳转型需求的存量客户。

### 1.2 多维度筛选

支持按以下维度对地图数据进行灵活筛选：

- **绿色行业支持维度** — 依据《绿色金融支持项目目录》分类筛选（节能环保、清洁生产、清洁能源、生态环境、基础设施绿色升级、绿色服务等）
- **重点行业领域节能维度** — 聚焦八大高碳行业清单（火电、钢铁、有色、石化、化工、建材、造纸、航空），筛选具有节能降碳改造需求的存量客户
- **转型行业支持维度** — 依据《转型金融支持目录》，筛选符合转型金融标准的项目和企业

### 1.3 企业业务分析报告

点击地图上的企业标记点，可调阅该企业的**业务分析报告**，包括：企业基本信息、行业归属、经营概况、绿色/转型属性判断、已合作绿色金融产品、潜在业务机会等，为一线业务人员提供即时的客户洞察。

## 二、潜在客户业务营销

面向绿色和转型两类潜在客户，提供清单管理、智能分析、营销赋能的一站式工具。核心能力由 **企业绿色低碳业务分析 Agent** 驱动，通过查询内外部数据自动生成业务分析报告。

### 2.1 转型潜在客户清单及业务分析

**新增客户洞察与分析功能**，支持业务人员通过查询企业基本信息，调用多维度模型进行自动评估：

- **数据维度**：企业信用资质、经营稳定性、碳排放强度、行业转型路径、政策支持力度等
- **输出内容**：自动生成《企业转型融资营销要点提示报告》，涵盖企业转型需求分析、推荐金融产品匹配、营销策略建议
- **导出能力**：支持导出 PDF 报告文件，供线下沟通和内部评审使用

### 2.2 绿色潜在客户清单及业务分析

系统接入绿色企业清单数据，以列表形式展示纳入绿色企业范围的企业名录及绿色潜在客户分类。主要功能：

- **清单管理**：列表展示绿色企业清单，标注绿色分类等级和潜在客户标签
- **关注管理**：业务人员可对企业添加"关注"或取消"关注"，形成个人绿色客户跟踪池
- **评价报告**：点击企业可查看《绿色潜在企业评价报告》明细，全面了解企业绿色属性
- **营销活动**：支持对清单内企业发起营销活动，记录营销触达和跟进状态

### 2.3 企业绿色低碳业务分析（Agent）

**新增客户洞察与分析功能**，由智能 Agent 驱动：

- 业务人员输入企业名称或统一社会信用代码，系统自动调取企业**信用资质、经营稳定性、绿色潜力**等多维度信息
- 调用绿色模型进行综合评估，自动生成《绿色业务分析报告》
- 报告内容涵盖：绿色产业分类匹配、绿色金融产品推荐、融资场景分析、风险提示
- 支持**导出 PDF 报告文件**，便于存档和跨部门流转

### 2.4 营销过程管理

营销过程管理是贯穿潜在客户挖掘到业务落地转化的全流程跟踪模块，为业务人员提供标准化的营销节点管理和进度可视化能力。

**核心流程**：

```
客户识别 → 初步对接 → 需求分析 → 方案提报 → 签约落地 → 持续跟进
```

**功能要点**：

- **营销阶段管理**：系统预设标准化营销阶段（初步对接、需求分析、方案提报、签约落地等），业务人员可按阶段推进并记录每个节点的对接人、沟通内容、下一步计划，确保营销过程有据可查
- **进度可视化**：以时间倒序展示营销过程记录，清晰呈现每个客户的营销推进轨迹，支持管理层快速掌握团队整体的营销动态
- **企业信息联动**：自动关联企业基本信息（行业分类、经营状况、绿色/转型属性等），业务人员在营销过程中无需切换页面即可查看完整的客户画像
- **客户分类跟踪**：支持按绿色客户、转型客户、园区项目客户等维度分类管理营销进程，分别匹配差异化的金融产品和营销策略
- **状态标记与提醒**：支持对重点关注客户进行标记，设置下次跟进提醒，避免商机遗漏

营销过程管理模块同时服务于**客户级**（潜在客户营销过程管理）和**项目级**（项目营销过程管理）两类场景，前者聚焦企业客户的触达与转化，后者聚焦园区及工程项目的对接与推进，两者共享统一的阶段管理框架，确保全行营销工作的标准化与可量化。

## 三、低碳转型项目营销

聚焦低碳转型领域的项目级营销管理，覆盖零碳/低碳园区和碳排放双控两大专题场景，建立从项目发现到方案输出的完整营销闭环。

### 3.1 零碳/低碳园区项目清单

围绕零碳/低碳园区建设需求，分阶段、分类型、分服务主体展示项目需求清单与对接情况。

**阶段维度**：
- 低碳园区 — 已启动节能改造或能源结构优化的园区
- 近零碳园区 — 碳排放强度显著下降、接近零碳目标的园区
- 零碳园区 — 已实现或规划实现碳中和的园区

**八大覆盖场景**：

| 序号 | 场景 | 说明 |
|------|------|------|
| 1 | 用能结构转型 | 光伏、风电、储能、氢能等清洁能源替代 |
| 2 | 节能降碳 | 工业节能改造、建筑节能、综合能源服务 |
| 3 | 优化产业结构 | 绿色产业链构建、低碳产业导入 |
| 4 | 资源节约集约 | 节水、节地、节材技术改造 |
| 5 | 完善基础设施 | 绿色交通、智慧能源管理平台 |
| 6 | 先进适用技术 | 低碳技术研发与应用推广 |
| 7 | 提升能碳管理能力 | 碳排放监测、碳资产管理 |
| 8 | 改革创新 | 碳金融、绿电交易等机制创新 |

**核心功能**：
- 按园区类型（经济技术开发区、高新技术产业开发区、自贸区等）和服务主体分类筛选
- 查看**潜在项目库**，了解项目规模、融资需求、对接状态
- 查看项目**业务营销方案**，为一线提供可执行的营销指导

### 3.2 "碳排放双控"专题项目清单

对接碳排放总量和强度"双控"政策要求，分类展示新增项目需求：

| 专题方向 | 聚焦内容 | 对接监管 |
|----------|----------|----------|
| **能源转型** | 节能降碳项目 | 能源局监管要求 |
| **产业升级** | 绿色工厂建设 | 经信局工作要求 |
| **全面节约** | 资源集约利用项目 | 发改委导向 |
| **"五碳"工程** | 源头减碳、过程降碳、末端固碳、碳资产开发、碳金融服务 | 多部门协同 |

每个专题方向下均支持查看潜在项目库和业务营销方案。

## 五、绿色识别业务管理

绿色识别是系统的核心业务模块，覆盖合同绿色认定、贷款预识别、反洗绿复核三大业务场景。模块由智能 Agent 驱动，调用绿色智能识别模型（基于大模型语义分析）完成自动化识别。

> **安全提示**：绿色智能识别模型的推理计算全部在行内服务器完成，融资用途、客户信息等敏感数据**不传输至外部**。

### 5.1 合同绿色认定

合同绿色认定是贷后绿色资产认定的核心流程，实现从任务生成到认定完成的自动化闭环。

**任务生成机制**：

- **自动生成**：系统通过数据仓库日终批量获取全量合同数据，根据"业务品种维护功能"中设定的需要进行绿色认定的业务品种，筛选生效状态的合同，自动生成绿色认定待办任务
- **手工生成**：支持输入其他业务品种或其他状态下的合同编号，手工生成绿色认定待办任务
- **待办同步**：待办任务生成后，同步至行内统一待办系统，业务人员可在统一待办中处理

**认定执行流程**：

1. **贷款类型判断** — 系统自动判断合同贷款类型
2. **项目类合同** — 引导业务人员选择关联项目绿色认定结果，或重新调用绿色智能识别模型
3. **非项目类合同** — 直接调用绿色智能识别模型进行智能识别
4. **结果展示** — 系统依据《绿色金融支持项目目录》（2025 版），展示绿色分类目录匹配结果、模型识别分析结果、资金用途等业务填报建议
5. **佐证材料** — 业务人员选定业务所属绿色分类后，系统自动提示该分类对应的绿色佐证材料提交建议，支持上传并保存佐证材料

### 5.2 贷款预识别

贷款预识别是信贷业务**放款前**的绿色金融前置环节，为正式绿色认定提供智能参考。

**操作流程**：

1. 业务人员手工录入客户基本信息（国民经济行业、地区、主营业务等）
2. 填写合同融资用途描述
3. 点击"开始识别"按钮
4. 系统依据《绿色金融支持项目目录（2025 版）》调用大模型对融资用途进行语义分析
5. 自动匹配并返回**排名前三**的绿色分类结果
6. 展示建议的佐证材料清单

预识别结果不作为最终认定结论，但为业务人员在贷前阶段判断绿色属性、准备认定材料提供高效参考。

### 5.3 "反洗绿"模型智能识别

"反洗绿"（Anti-Greenwashing）是绿色金融质量控制的关键环节，用于防范"漂绿""洗绿"风险——即企业将非绿色项目包装为绿色项目以获取绿色信贷优惠的行为。

**功能说明**：

- 授权范围：省分行超级管理员可调用"反洗绿"模型
- 功能逻辑：对已认定的合同进行智能识别结果复核验证，模型从多维度交叉校验认定结论的合理性
- 监控价值：监控全行绿色/转型资产质量，及时发现和纠正可能的认定偏差
- 结果处理：复核结果异常的任务可退回重新认定，形成质量控制闭环

## 六、系统管理

系统管理模块为"至简绿达"提供基础的用户权限、组织架构和数据标准支撑，确保系统在行内多层级组织体系下安全、规范运行。

| 子模块 | 功能概述 |
|--------|----------|
| **用户管理** | 支持用户信息的增删改查，可关联角色分配权限，控制用户账号的启用/停用状态，保障系统访问安全 |
| **角色管理** | 定义系统角色并分配菜单/按钮权限及数据权限范围（全部/本机构及下设/本机构/本人），角色可关联多个用户，实现权限的批量授予 |
| **机构管理** | 以树形结构维护组织架构（总行→分行→支行），支持机构的层级管理、状态控制及数据导出 |
| **数据字典** | 维护系统基础数据项（绿色产业分类、行业代码、融资用途等枚举值），支持树形层级管理、批量导入导出，为业务下拉选项提供标准化数据源 |

## 七、系统架构与安全

| 维度 | 说明 |
|------|------|
| **部署架构** | 行内部署，所有应用服务、数据库、模型推理均在行内服务器运行 |
| **数据安全** | 数据不出行，客户信息、融资用途等敏感数据不传输至外部网络 |
| **模型安全** | 绿色智能识别大模型采用本地化部署，推理计算在行内完成 |
| **权限安全** | 基于 RBAC 的细粒度权限控制，支持数据权限按机构层级隔离（全部/本机构及下设/本机构/本人） |
| **审计追溯** | 认定结论、模型识别结果、操作日志全程留痕，满足监管审计要求 |
